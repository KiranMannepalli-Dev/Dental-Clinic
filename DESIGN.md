# DESIGN.md

## Design Philosophy

Build interfaces that feel premium, calm, minimal, balanced, and highly usable.
Every section, component, interaction, and layout should prioritize:

* Clarity over decoration
* Consistency over experimentation
* Readability over visual effects
* Simplicity over complexity
* Functionality over unnecessary animations
* Balance over excessive spacing or compression
* Professional modern aesthetics over trendy UI styles

The UI should feel smooth, lightweight, elegant, and production-grade.

---

# Core UI/UX Principles

## 1. Visual Consistency

Consistency is mandatory across the entire application.

Maintain consistency in:

* Typography
* Spacing
* Layout structure
* Component sizing
* Button styles
* Card styles
* Border radius
* Hover interactions
* Colors
* Input fields
* Section spacing
* Alignment systems
* Grid structures
* Animation timing

Users should never feel like different sections belong to different products.

---

## 2. Minimal Premium Design

Avoid excessive visual styling.

Do NOT use:

* Heavy shadows
* Floating glassmorphism
* Overly rounded cards
* Glossy effects
* Neon colors
* Excessive borders
* Unnecessary animations
* Over-decorated sections
* Large floating UI elements

Instead use:

* Clean spacing
* Soft contrast
* Sharp readability
* Balanced layouts
* Subtle hover states
* Professional typography
* Minimal depth
* Structured hierarchy

The design should feel modern and premium without trying too hard.

---

# Typography System

## Typography Philosophy

Typography should feel elegant, readable, balanced, and structured.

Do not rely on boldness to create hierarchy.

Hierarchy should come from:

* Font size
* Spacing
* Layout structure
* Alignment
* Visual grouping
* Contrast
* Consistent scale systems

---

## Font Weight Rules

Avoid using bold text everywhere.

### Use font weights carefully:

| Usage              | Weight  |
| ------------------ | ------- |
| Main Hero Headings | 600     |
| Section Headings   | 500–600 |
| Card Titles        | 500     |
| Body Text          | 400     |
| Small Labels       | 400–500 |
| Captions           | 400     |

Never overuse:

* 700+
* Extra bold fonts
* Black font weights

Too much boldness destroys premium aesthetics.

---

## Font Family Rules

Use clean modern font families.

Recommended:

* Inter
* Manrope
* Geist
* General Sans
* Satoshi
* Plus Jakarta Sans

Avoid decorative or playful fonts.

Typography should feel:

* Neutral
* Professional
* Highly readable
* Modern
* Minimal

---

## Typography Scale

Maintain a balanced typography scale.

### Example Scale

| Type            | Size    |
| --------------- | ------- |
| Hero Heading    | 48–64px |
| Section Heading | 28–36px |
| Subheading      | 20–24px |
| Card Title      | 16–18px |
| Body Text       | 14–16px |
| Small Text      | 12–13px |

Avoid large jumps in font sizing.

Maintain smooth visual hierarchy.

---

## Readability Rules

Never reduce text opacity excessively.

Avoid:

* Low contrast text
* Washed-out typography
* Extremely thin font weights
* Tiny unreadable labels

Text should always remain:

* Clear
* Sharp
* Readable
* Accessible
* High visibility

Maintain strong readability across all devices.

---

# Color System

## Color Philosophy

Use limited colors.

Follow the:

# 60–20–10–10 Rule

| Usage                 | Percentage |
| --------------------- | ---------- |
| Primary Background    | 60%        |
| Secondary Surface     | 20%        |
| Accent Color          | 10%        |
| Interactive/CTA Color | 10%        |

---

## Color Usage Rules

Use:

* Soft backgrounds
* Deep readable text colors
* Calm neutral palettes
* Professional contrast systems

Avoid:

* Harsh saturation
* Neon colors
* Loud gradients
* Rainbow palettes
* Excessive color usage

---

## Recommended Style

### Backgrounds

Use soft tones like:

* Off-white
* Light gray
* Soft beige
* Muted dark tones
* Slight warm neutrals

### Text Colors

Use:

* Deep charcoal
* Neutral black
* Muted navy
* Professional dark gray

Avoid pure black everywhere.

---

## Gradient Rules

Do NOT use gradients unless absolutely necessary.

Default system should use:

* Solid colors
* Flat backgrounds
* Minimal visual noise

Gradients reduce premium consistency when overused.

---

# Layout System

## Layout Philosophy

Layouts should feel balanced, breathable, and structured.

Avoid:

* Uneven spacing
* Random padding
* Oversized empty areas
* Cramped sections

Use consistent spacing systems.

---

## Screen Edge Rules

Maintain equal left and right spacing across the application.

### Example

Desktop container padding:

* 80px–120px

Tablet:

* 40px–60px

Mobile:

* 16px–24px

Never allow content to touch screen edges.

---

## Section Spacing

Use balanced vertical rhythm.

### Recommended Section Gaps

| Usage               | Gap      |
| ------------------- | -------- |
| Large Sections      | 80–120px |
| Internal Components | 24–40px  |
| Card Content        | 16–24px  |
| Small Groups        | 8–16px   |

Spacing should feel intentional and mathematically consistent.

---

## Grid System

Use structured grid layouts.

Preferred:

* 12-column responsive grid
* Equal spacing systems
* Predictable alignment
* Consistent breakpoints

Avoid random widths and inconsistent alignment.

---

# Cards System

## Card Philosophy

Cards should feel subtle, premium, lightweight, and structured.

---

## Card Radius Rules

Do NOT use highly rounded cards.

### Standard Radius

* 6px border radius

Maintain the same radius across:

* Cards
* Buttons
* Inputs
* Dropdowns
* Modals

Consistency creates professionalism.

---

## Card Styling Rules

Avoid:

* Heavy shadows
* Floating glassmorphism
* Over-styled borders
* Excessive layering

Use:

* Thin subtle borders
* Soft surface differentiation
* Minimal depth
* Clean separation

Cards should integrate naturally into layouts.

---

## Hover States

Animations should be subtle.

Allowed:

* Slight background change
* Small border transition
* Minimal elevation
* Soft opacity transition

Avoid:

* Bounce animations
* Large scaling
* Aggressive motion
* Complex transforms

Animation should support usability, not distract from it.

---

# Button System

## Button Philosophy

Buttons should feel clear, strong, minimal, and usable.

---

## Button Styling Rules

Use:

* Solid colors
* Minimal styling
* Medium font weights
* Clear padding
* Consistent heights

Avoid:

* Pill-shaped buttons
* Oversized buttons
* Gradient buttons
* Heavy glow effects

---

## Button Radius

Standard:

* 6px radius

Maintain consistency with all interactive components.

---

## Button Heights

| Type      | Height  |
| --------- | ------- |
| Primary   | 44–48px |
| Secondary | 40–44px |
| Small     | 32–36px |

---

# Pills, Tags, Labels

## Philosophy

Small UI elements should remain minimal and context-oriented.

Avoid oversized pills.

Use:

* Compact sizing
* Balanced padding
* Minimal font sizes
* Contextual colors

Pills should support content, not dominate it.

---

# Forms & Inputs

## Input Design Rules

Inputs should prioritize usability and clarity.

Use:

* Consistent heights
* Clear borders
* Good padding
* Readable placeholder text
* Strong focus states

Avoid:

* Fancy borders
* Glow effects
* Large shadows
* Over-animated inputs

---

## Input Radius

* 6px

Keep consistent with the entire design system.

---

## Form UX Rules

Forms should:

* Minimize friction
* Reduce cognitive load
* Clearly indicate errors
* Provide instant validation feedback
* Maintain spacing consistency

---

# Loading States

## Loading Philosophy

Loading states should feel smooth and lightweight.

Avoid:

* Large spinners everywhere
* Flashing animations
* Over-designed skeletons

Use:

* Minimal skeleton loaders
* Soft shimmer effects
* Small inline progress indicators

Loading should feel calm and professional.

---

# Error Handling UI

## Error UX Principles

Error handling should remain simple and informative.

Avoid:

* Aggressive red colors
* Technical jargon
* Large disruptive modals

Use:

* Clear messages
* Small inline notifications
* Helpful guidance
* Calm visual feedback

Users should understand:

* What happened
* Why it happened
* What to do next

---

# Empty States

Empty states should feel purposeful.

Include:

* Simple illustrations or icons
* Clear messaging
* Helpful CTA
* Minimal visual clutter

Avoid dead-end screens.

---

# Animation System

## Motion Philosophy

Motion should support usability.

Not decoration.

---

## Animation Rules

Use:

* 150ms–300ms transitions
* Smooth easing
* Minimal movement

Avoid:

* Large parallax
* Continuous floating animations
* Dramatic entrance effects
* Excessive motion

Interfaces should feel stable and controlled.

---

# Responsiveness

## Responsive Philosophy

Every UI must work seamlessly across:

* Desktop
* Laptop
* Tablet
* Mobile

---

## Responsive Rules

Maintain:

* Consistent spacing ratios
* Readable typography
* Touch-friendly interactions
* Proper stacking behavior

Avoid broken layouts or compressed content.

---

# Accessibility

## Accessibility Standards

Maintain accessibility by default.

Include:

* Proper contrast ratios
* Keyboard navigation
* Focus visibility
* Readable font sizes
* Semantic HTML
* Screen reader support

Accessibility is part of premium UX.

---

# Performance & Fast Loading

## Performance Philosophy

Fast interfaces feel premium.

Performance is part of design quality.

---

## Performance Rules

Optimize:

* Image sizes
* Font loading
* Bundle size
* Lazy loading
* Code splitting
* Rendering efficiency

Avoid:

* Heavy animations
* Excessive libraries
* Large media assets
* Unnecessary API calls

---

## Image Optimization

Use:

* WebP/AVIF formats
* Responsive images
* Lazy loading
* Proper compression

Never use oversized images.

---

# Navigation UX

## Navigation Principles

Navigation should feel obvious and frictionless.

Use:

* Clear hierarchy
* Predictable structure
* Minimal nav items
* Proper spacing
* Consistent active states

Avoid cluttered navigation bars.

---

# Micro Interactions

Micro interactions should feel subtle and functional.

Examples:

* Hover feedback
* Button press states
* Form validation feedback
* Success indicators
* Smooth dropdown opening

Avoid excessive animated effects.

---

# Design Consistency Checklist

Before finalizing any UI:

* Are spacing values consistent?
* Are font sizes balanced?
* Is hierarchy visually clear?
* Are colors limited and controlled?
* Are hover states subtle?
* Are components reusable?
* Is readability strong?
* Is responsiveness handled?
* Is accessibility maintained?
* Is performance optimized?
* Are animations minimal?
* Is the interface visually calm?

---

# Final Design Standard

The final product should feel:

* Minimal
* Premium
* Professional
* Structured
* Fast
* Readable
* Calm
* Consistent
* Modern
* Production-grade

Every design decision must support usability, clarity, performance, and visual balance.
